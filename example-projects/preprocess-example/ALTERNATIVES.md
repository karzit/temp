# 대체 가능한 기술 (참고용)

이 프로젝트는 `langchain` + `PyMuPDF` + `opensearch-py` 조합으로 만들어졌다.
아래는 같은 역할을 대신할 수 있는 다른 선택지들을 간단히 소개하는 문서다.
실습 코드는 그대로 두고, "이럴 때는 이런 대안도 있다" 정도만 참고하면 된다.

## 오케스트레이션 (langchain)

| 대안 | 언제 고려하나 |
|---|---|
| `LlamaIndex` | 청킹/인덱싱처럼 RAG에 특화된 작업 위주라면 더 단순하게 짤 수 있음 |
| 직접 구현 (라이브러리 없이) | 파이프라인이 단순해서 프레임워크 오버헤드가 부담될 때 |

## PDF 텍스트 추출 (PyMuPDF)

| 대안 | 언제 고려하나 |
|---|---|
| `pypdf` | 가볍게 텍스트만 뽑으면 될 때 (이미 rag-regulation-example에서 사용 중) |
| `pdfplumber` | 표(table) 구조를 유지하며 추출해야 할 때 |
| `unstructured` | PDF 외에 다양한 포맷(docx, html 등)까지 한 번에 처리하고 싶을 때 |
| `Docling` (IBM) | 레이아웃 인식 품질이 중요한 복잡한 문서일 때 |

## 벡터 저장/검색 (opensearch-py)

| 대안 | 언제 고려하나 |
|---|---|
| `pgvector` | 이미 PostgreSQL을 쓰고 있으니(원본 저장용) 별도 벡터DB 없이 통합하고 싶을 때 |
| `Qdrant` / `Weaviate` / `Milvus` | 벡터 검색 성능/기능(필터링, 하이브리드 검색 등)이 중요한 대규모 서비스일 때 |
| `Chroma` | 로컬/소규모 실습 환경에서 가볍게 띄우고 싶을 때 |

## 참고

- 이 문서는 실습 편의를 위한 안내이며, 실제 코드 마이그레이션은 다루지 않는다.
